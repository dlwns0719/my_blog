1 1. 프롬프트 기초

1.1 1.1 인간-컴퓨터 상호작용의 변화

  • 문자의 발명 → 인류의 기록 정확도 향상
  • 인쇄술 → 지식 대중화
  • 전화·라디오·TV → 즉각적인 원격 소통
  • 인터넷·모바일 → 실시간 대화
  • 생성형 AI → 인간-기계 상호작용 고도화

1.2 1.2 프롬프트의 정의

  • Promptus(나타내다/생산하다)에서 유래
  • 현재 의미: 행동을 유도하는 신호
  • AI에서: 모델에게 답변을 생성하도록 입력하는 문장

1.3 1.3 프롬프트 vs 프롬프트 엔지니어링

항목 프롬프트 프롬프트 엔지니어링
목적 응답 생성 요청 원하는 형태로 정확한 응답을 얻기 위한 설계
필요성 기본 입력 모델 작동 방식·언어적 맥락 이해 필요
예시 “용에 대한 이야기 해줘” “용감한 용이 마을을 구하는 5문장 이야기 생성해줘. 대화 포함.”

2 2. 대형 언어 모델(LLM) 이해

2.1 2.1 개념

  • 방대한 데이터 기반 사전학습 모델
  • Autoregression: 이전 토큰 기반으로 다음 단어 예측
  • 기반 기술: Transformer 구조(셀프 어텐션)

2.2 2.2 주요 모델 패밀리

  • OpenAI – GPT-4o: 멀티모달·실시간 처리
  • Anthropic – Claude 3: 안전성 중점, 헌법적 AI
  • Google – Gemini / BERT / T5: 다중 모달 처리

2.3 2.3 모델 튜닝 방식

  • Fine-Tuning
  • Prompt Engineering
  • Feature-Tuning(특징 공간 해석 기반 제어)

3 3. 프롬프트 기본 요소

3.1 3.1 필수 4요소

  1. Instruction: 어떤 작업을 해야 하는가
  2. Context: 배경 정보
  3. Input Data: 처리 대상 데이터
  4. Output Indicator: 출력 형식 규정

3.2 3.2 프롬프트 타입 구조(Type A–D)

  • A: 지시문 + 출력문
  • B: 지시문 + 맥락 + 출력문
  • C: 지시문 + 맥락 + 예시 + 출력문
  • D: 지시문 + 입력값 + 출력문

예시와 함께 구조적으로 프롬프트를 설계하면 LLM이 더 안정적으로 원하는 출력 생성.

4 4. 프롬프트 설계 원칙

4.1 4.1 4대 원칙

  1. 범용성: 다양한 상황에서도 동작
  2. 목적 지향성: 특정 사용자 목적 반영
  3. 일관성: 반복 실행해도 동일 품질
  4. 경제성: API 비용 절감

4.2 4.2 Scaffolding Prompting

  • 인간의 학습 이론(ZPD)을 적용
  • Understand → Ideation → Analysis → Apply → Evaluate → Generalize
  • 단계적 사고구조 유도

4.3 4.3 핵심 5 Tips

4.3.1 Tip1. 최신 모델 사용

4.3.2 Tip2. 명확한 동사로 지시

예: 분석해, 정리해, 요약해, 작성해 등

4.3.3 Tip 3. 프롬프트를 구조화하자

프롬프트를 구조화하면 LLM이 입력을 더 명확하게 해석할 수 있고,
작업 목적·규칙·입력 데이터·출력 형식을 혼동 없이 처리할 수 있다.
구조화를 위한 핵심 기법은 Markdown, 코드 프롬프트, XML 태그이다.

4.4 3.1 Markdown 구조화 (Headers, Lists, Emphasis)

Markdown은 LLM이 가장 잘 해석하는 구조적 문법 중 하나이다.
섹션을 명확히 나누고, 각 부분을 시각적으로 구분해줌으로써
모델의 오류를 줄이고 반복 작업 시 유지보수성이 높아진다.

4.4.1 1) 헤더(Headers) 활용

헤더는 프롬프트의 구성을 크게 나누는 데 사용한다.

# 역할(Role)
너는 스포츠 기사 분석 엔진이다.

## 작업(Task)
아래 텍스트에서 장소(도시, 국가, 축구 클럽)를 모두 추출해라.

## 출력 형식(Format)
- "장소: A, B, C" 형태의 한 줄 출력
- 설명 금지

## 텍스트(Text)
{{TEXT}}

4.4.2 2) 목록(Lists) 활용

절차, 규칙, 조건을 번호 목록으로 정리하면 LLM이 하나씩 적용하기 쉽다.

다음 규칙을 따라 작업을 수행해라.
1. 사람 이름은 제외한다.
2. 축구 클럽은 장소로 포함한다.
3. 결과는 쉼표로 구분한다.
4. 설명 문장은 쓰지 않는다.

4.4.3 3) 강조(Emphasis) 사용

중요 조건을 굵게 또는 기울임으로 표시해 모델의 우선순위를 높인다.

- 반드시 **한국어**로 작성하라.
- 출력은 **한 줄**로만 작성한다.
- *설명 문장은 포함하지 않는다.*

4.5 3.2 코드 프롬프트(Code Prompting)

자연어 대신 코드 형식으로 의도를 명세하면 애매함이 줄어들고 LLM이 구조적으로 이해해 할루시네이션이 감소한다.

# 역할 정의
role = "정보 추출 엔진"

# 작업 정의
task = "텍스트에서 '장소명'을 추출해 정렬된 리스트로 반환"

# 출력 형식 정의
output = {
    "places": ["string"]
}

코드 방식의 장점:

  • 논리 구조가 명확하게 표현됨
  • 조건과 출력 형식이 혼동되지 않음
  • 정형화된 포맷을 자주 재사용 가능

4.6 3.3 XML 태그 기반 구조화 (Claude 최적화)

XML 태그는 텍스트 내 특정 영역을 명확히 구분해 Claude뿐 아니라 대부분의 LLM에서 구조적 파싱을 돕는 강력한 방식이다.

4.6.1 1) 태그로 영역 지정

<email>
Show up at 6AM because I say so.
</email>

Make this email more polite.

LLM은 <email>...</email> 내부 텍스트만 변환하면 된다는 것을 명확히 인식한다.

4.6.2 2) 데이터 템플릿 예시

<animal>{{ANIMAL}}</animal>

입력만 바꿔 여러 데이터를 동일한 템플릿에 적용할 수 있다.

예:

<animal>Cow</animal>
<animal>Dog</animal>
<animal>Seal</animal>

장점:

  • 구역별 의미가 명확
  • 반복 작업에서 수정이 쉽고 안정적
  • 모델이 태그 단위로 파싱하기 때문에 오류 감소

4.7 3.4 구조화 기법이 필요한 이유

  • 명령어, 입력 데이터, 출력 형식을 명확히 구분해 모델의 해석 오류를 줄인다.
  • 규칙을 시각적으로 구분하여 작업 수행 품질의 일관성과 재현성을 높인다.
  • 재사용 가능한 템플릿이 되어 프롬프트 제작 비용과 시간 절감에 기여한다.

4.8 3.5 구조화 적용 사례 (장소명 추출)

# 역할(Role)
너는 텍스트에서 장소명을 추출하는 엔진이다.

## 작업(Task)
텍스트 내 도시, 국가, 축구 클럽 이름을 모두 "장소"로 간주하고 추출해라.

## 출력 형식(Format)
- 한 줄로만 출력한다.
- 형식: `장소: A, B, C`
- 설명 문장 금지.

## 텍스트(Text)
음바페는 26일 쿠프 드 프랑스 프랑스컵 올랭피크 리옹과의
결승전에서 풀타임을 뛰며 2-1로 팀 승리를 도왔다.
3년 만에 대회 정상에 오른 PSG는 역대 최다 15회 우승으로
2위 마르세유(10회)를 멀찍이 따돌렸다.
음바페는 이날 PSG에서의 커리어를 마무리했다.

4.8.1 Tip4. 단문 중심

  • 한국어는 짧고 명확하게 작성
  • 톤·형식·분량을 구체적으로 지시

4.8.2 Tip5. 하지 말 것(X) → 할 것(O)

  • 부정 지시보다 긍정적 행동 지시가 안정적

5 5. LLM의 한계와 프롬프트 엔지니어링 필요성

5.1 5.1 LLM 한계

  • 할루시네이션
  • 편향성
  • 근거 없는 출처 생성
  • 수학 및 추론 한계
  • Prompt Injection(프롬프트 해킹)

5.2 5.2 프롬프트 엔지니어링이 필요한 이유

  • 정확성 향상
  • 오류 및 편향 감소
  • 모델 행동 제어
  • 사용자 경험 개선
  • 비용 절감
